# PROMPT_SNIPPETS.md

> 这里是可以直接复制给 Codex 的提示模板。

---

## 1. 阅读模式

```text
先不要修改代码。

阅读以下目录并输出：
1. 主入口调用链
2. 每个核心类职责
3. 数据流路径
4. 线程/资源生命周期
5. 你认为最容易出 bug 的 5 个点

要求：
- 引用具体文件和函数名
- 不要泛泛而谈
- 不要提交任何代码修改
```

---

## 2. 受限实现模式

```text
先不要改代码。

任务目标：
[在这里写目标]

只允许修改：
- [file 1]
- [file 2]
- [file 3]

禁止修改：
- [module / path 1]
- [module / path 2]

先输出：
1. 你对任务的理解
2. 计划修改的文件
3. 不会修改的文件
4. 实现方案
5. 风险点
6. 验证方案

然后再提交最小 patch。
不要顺手重构。
不要扩大修改范围。
```

---

## 3. Diff 解释模式

```text
不要继续写代码，只解释当前 diff：

1. 改了哪些文件
2. 每个文件改了什么
3. 为什么这样设计
4. 调用链有没有变化
5. 生命周期 / 所有权有没有变化
6. 哪些地方我应该优先 review
7. 仍有哪些风险

要求：
- 结合具体函数名
- 面向维护者解释
- 不要只重复代码表面差异
```

---

## 4. Bug 修复模式

```text
不要直接修。

先做：
1. 复述 bug
2. 找最小复现
3. 列出根因候选并按概率排序
4. 写一个 failing test
5. 再做最小修复
6. 跑回归验证
7. 解释影响面

限制：
- 不顺手重构
- 不扩大公共接口
- 不修改无关模块
```

---

## 5. Review agent 模式

```text
假设你是 reviewer，不是作者。

请审查这次改动，输出：
1. 这次 patch 是否符合任务目标
2. 是否有范围失控
3. 是否有线程 / 生命周期 / 越界风险
4. 测试是否充分
5. 是否建议合并
6. 若不建议，最关键的 3 个问题是什么
```
